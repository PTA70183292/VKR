Требуется Python версии 3.10 для совместимости с стабильными версиями библиотек
    Docker >= 24.x
    Docker Compose v2
    Для Windows:
        Требуется WSL2 (wsl --install)
    Для Linux:
        Нативная поддержка Docker, WSL не требуется 
    Git

Если есть GPU Nvidia с поддержкой CUDA:
    Драйвер Nvidia (рекомендуется >= 535)
    NVIDIA Container Toolkit (nvidia-docker)
    Минимум 8 GB видеопамяти (рекомендуется 12–16 GB)

Если НЕТ GPU Nvidia - можно запустить проект только в режиме инференса, на CPU. Но только функция дообучения не будет работать. 
    Для этого - в docker-compose.yml закомментируйте эти строки: 
        "deploy:                                           
            resources:                                     
                reservations:                                
                devices:                                   
                    - driver: nvidia                         
                    count: 1                         
                    capabilities: [gpu]"
    Раскомментируйте:
    "environment:
      - INFERENCE_DEVICE=cpu"

Поднять контейнеры: docker compose up --build
    Автоматически загрузится предобученная модель (адаптер) из проекта блокнота  https://huggingface.co/PTA70183292/rubert-base-cased-conversational_8bit_r64_l256

Запустить фронтент (студент): http://localhost:8502/
Запустить фронент (сотрудники): http://localhost:8501/

Для обучения модели из другого датасета:
    Загрузить CSV файл
    Обязательные колонки в файле:
        text
        label
    Есть тестовый датасет для проверки: .\Тестовый датасет для обучения\Датасет для проверки обучения.csv"

Для проверки загрузки готовой модели (адаптера) из файла (zip) можно использовать папку: .\Тестовая модель(адаптер) для загрузки из файла\На базе QLoRA r64.zip

Для проведения теcтов: docker-compose exec backend pytest tests/ -v --disable-warnings 

Важное примечание по безопасности: Файл .env намеренно включен в репозиторий исключительно для удобства проверки работы проекта и демонстрации конфигурации. В реальной эксплуатации данный файл добавляется в .gitignore, а чувствительные данные (ключи, пароли БД) передаются через защищенные переменные окружения сервера (CI/CD secrets).
